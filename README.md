# Автосервис — Лабораторная работа №3 (Вариант 2)

Приложение для автоматизации учёта заказов на ремонт автомобилей.  
Разработано на C++/CLI с использованием Windows Forms и ADO.NET.

## Требования

- Windows 10/11
- Visual Studio 2022 с компонентом "Разработка классических приложений на C++"
- SQL Server (Express или полная версия) или LocalDB
- .NET Framework 4.7.2+

## Быстрый старт

### 1. Создание базы данных

Откройте SQL Server Management Studio (SSMS) и выполните скрипт:

```
sql/setup_database.sql
```

Скрипт создаст базу `AutoServiceDB` со всеми таблицами и тестовыми данными.

### 2. Настройка подключения

В файле `AutoService/app.config` измените строку подключения (`Server=...`), указав имя вашего SQL Server.

Примеры:
- `Server=localhost\SQLEXPRESS` — для SQL Server Express
- `Server=(localdb)\MSSQLLocalDB` — для LocalDB
- `Server=MYPC\SQLEXPRESS` — для именованного экземпляра

### 3. Сборка и запуск

1. Откройте `AutoService.sln` в Visual Studio 2022
2. Соберите проект (Ctrl+Shift+B)
3. Запустите (F5)

## Подключение к серверу БД

Приложение поддерживает два способа подключения:

### Windows аутентификация (Integrated Security)
- Галочка "Использовать Windows аутентификацию" включена
- Подключение идёт от имени текущего пользователя Windows
- Не нужно вводить логин/пароль SQL Server
- Строка: `Server=...;Database=AutoServiceDB;Integrated Security=true;`

### SQL Server аутентификация (логин/пароль)
- Галочка "Использовать Windows аутентификацию" выключена
- Нужно ввести логин и пароль SQL Server
- Строка: `Server=...;Database=AutoServiceDB;User Id=...;Password=...;`

## Тестовые учётные записи (вход в приложение)

| Логин      | Пароль   | Роль        |
|------------|----------|-------------|
| admin      | admin    | Admin       |
| client1    | client   | Client      |
| mechanic1  | mechanic | Mechanic    |
| manager1   | manager  | Manager     |

## Роли и возможности

### Admin
- Панель мониторинга (открытые заказы, загрузка механиков)
- Управление справочниками (услуги, запчасти) — CRUD
- Просмотр всех данных

### Client
- Просмотр своих заказов с цветовой индикацией статуса
- Создание нового заказа (выбор авто, услуг, дата)
- Уведомления в StatusStrip

### Mechanic
- Список своих заказов с фильтрацией по статусу
- Детали заказа: PropertyGrid автомобиля, ProgressBar ремонта
- Кнопки: «Начать работу», «Приостановить», «Завершить»

### Manager
- Все заказы с назначением механиков
- Изменение статуса заказов
- Отчёт по доходам за период с группировкой по услугам

## Безопасность

- Параметризованные запросы (защита от SQL-инъекций)
- Хеширование паролей (SHA2_256)
- Автоматический выход по таймауту неактивности (15 минут)
- Строки подключения во внешнем `app.config`
